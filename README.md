### // SYSTEMS_THINKING
- **Complexity:** I pull in native complexity (Kotlin/MethodChannels) when OS-level security is non-negotiable. I reject it when a simple in-memory sort can replace a complex DB query.
- **Abstractions:** If an abstraction hides the lifecycle of the underlying process, I avoid it. I prefer building my own bridges over using generic plugins.
- **Failures:** I treat production failures as mandatory system requirements for the next iteration.
